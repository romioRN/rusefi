# КРИТИЧЕСКОЕ ИСПРАВЛЕНИЕ: Преобразование BTDC в абсолютный угол для Pulse 1

## Найденная проблема

Таблица `secondInjectionAngleTable` содержит углы в **BTDC** (Before Top Dead Center - опережение впрыска), но код интерпретировал их как **абсолютные углы цикла**.

### Пример проблемы

Если таблица содержит **100°**, это означает:
- **100° BTDC** = впрыск на 100° ДО верхней мертвой точки
- В абсолютной системе 0-720°: **720° - 100° = 620°**

Но код возвращал просто **100°**, что было **НЕПРАВИЛЬНО**!

### Подтверждение из конфига

```ini
upDownLabel = "(later) Angle BTDC (sooner)"
```

Это ясно указывает что таблица содержит **BTDC углы**, а не абсолютные!

## Решение

Добавлено преобразование из BTDC в абсолютный угол:

```cpp
// Было (НЕПРАВИЛЬНО):
float baseAngle = interpolate3d(...secondInjectionAngleTable...);
// baseAngle = 100° ← интерпретировано как абсолютный угол!

// Стало (ПРАВИЛЬНО):
float btdcAngle = interpolate3d(...secondInjectionAngleTable...);  // 100° BTDC
float baseAngle = 720.0f - btdcAngle;  // Преобразовать в абсолютный: 620°
```

## Математика

### Система координат RUSEFI

```
0°      180°    360°    540°    720°
|       |       |       |       |
TDC     BDC     TDC     BDC     TDC
(Cyl1)          (Cyl2)          (Cyl3)
```

### Преобразование BTDC → Абсолютный угол

```
BTDC value = X° (опережение перед TDC)
Absolute angle = 720° - X°

Примеры:
- 0° BTDC (в TDC) → 720° абсолютно (конец цикла) → normalizeAngle → 0°
- 100° BTDC (раньше TDC) → 620° абсолютно
- 360° BTDC (за 360°) → 360° абсолютно
- 720° BTDC (за весь цикл) → 0° абсолютно
```

## Влияние на логи

### Было (НЕПРАВИЛЬНО):
```
Pulse 1: angle=100.0°  ← Неправильно! Это не то место впрыска
```

### Стало (ПРАВИЛЬНО):
```
Pulse 1: angle=620.0°  ← Правильно! Это 100° перед следующей TDC
```

Это означает Pulse 1 будет происходить в конце цикла (перед следующей TDC), что логично для второго впрыска!

## Отличие от Pulse 0

### Pulse 0 (обычный впрыск):

Таблица `injectionPhase` ТАКЖЕ содержит BTDC, но код это уже правильно обрабатывает:

```cpp
angle_t openingAngle = injectionOffset - injectionDurationAngle;
// injectionOffset из таблицы это КОНЕЦ впрыска в BTDC
// Вычитаем длительность чтобы получить НАЧАЛО
// Результат уже в правильной системе координат
```

### Pulse 1 (второй впрыск):

Раньше не было преобразования BTDC → абсолютный угол.
Теперь добавлено:

```cpp
float btdcAngle = interpolate3d(...);  // Из таблицы (BTDC)
float baseAngle = 720.0f - btdcAngle;  // Преобразовать в абсолютный
// Дальше применяется timing mode correction как обычно
```

## Timing Mode Correction

После преобразования применяется timing mode correction точно так же как для Pulse 0:

- **START mode:** angle не меняется (это начало впрыска)
- **CENTER mode:** сдвиг на -duration/2 (чтобы впрыск был центрирован)
- **END mode:** сдвиг на -duration (чтобы впрыск заканчивался в указанный момент)

## Валидация преобразования

После преобразования все углы нормализуются функцией `normalizeAngle()`:

```cpp
return normalizeAngle(correctedAngle);
// Нормализует в диапазон 0-720°
```

Это гарантирует что результат всегда в правильном диапазоне.

## Тестирование

После этого исправления логи должны показать:

```
Pulse 0: angle=534.0°  (остается как было)
Pulse 1: angle=620.0°  (было 100°, теперь правильно 720-100=620°)

Dwell = 620 - (534 + 0.8) = 85.2°  (вместо 285.2°)
```

Это намного более разумный dwell между впрысками!
