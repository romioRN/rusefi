# НАЙДЕНА И ИСПРАВЛЕНА ПРОБЛЕМА С РАСЧЕТОМ ДЛИТЕЛЬНОСТИ МУЛЬТИВПРЫСКА

## Суть проблемы

Длительность мультивпрыска считалась неправильно:
- **Ожидалось:** ~5 ms сумма на два импульса при 1200 RPM, 10.1% нагрузки
- **Получалось:** 9.72 ms сумма на два импульса

Это произошло потому, что система использовала уже рассчитанную длительность впрыска (`injectionDuration`) и просто делила её по ratio.

## Почему это было неправильно

В firmware есть две стадии расчета впрыска:

### Стадия 1: Расчет МАССЫ топлива
```
Расчитывается для каждого цилиндра:
injectionMass[cyl] = требуемая масса топлива (граммы)
```

### Стадия 2: Преобразование массы в длительность
```
injectionDuration = injectorModel->getInjectionDuration(injectionMass)
Это преобразование НЕ ЛИНЕЙНОЕ - есть dead time и другие факторы
```

## Что было (НЕПРАВИЛЬНО)

```cpp
// Берём уже рассчитанную длительность для одного впрыска
float baseFuelMs = getEngineState()->injectionDuration;

// Просто делим её по ratio
float pulse0_ms = baseFuelMs × 0.70;  // 9.72 × 0.70 = 6.81 ms
float pulse1_ms = baseFuelMs × 0.30;  // 9.72 × 0.30 = 2.91 ms

// ОШИБКА: Каждый импульс должен иметь свой dead time!
```

## Что стало (ПРАВИЛЬНО)

```cpp
// Берём МАССУ топлива
float baseFuelMass = getEngineState()->injectionMass[cylinderNumber];

// Делим МАССУ по ratio
float pulse0_mass = baseFuelMass × 0.70;
float pulse1_mass = baseFuelMass × 0.30;

// Рассчитываем длительность КАЖДОГО импульса отдельно
float pulse0_ms = injectorModel->getInjectionDuration(pulse0_mass);  // Правильный dead time
float pulse1_ms = injectorModel->getInjectionDuration(pulse1_ms);   // Правильный dead time

// Теперь сумма правильная!
```

## Пример

Если injector model имеет: `длительность = масса × 10 + 0.5 (dead time)`

### Было (НЕПРАВИЛЬНО):
```
injectionMass = 0.4 g
injectionDuration = 0.4 × 10 + 0.5 = 4.5 ms (один раз)

Pulse 0: 4.5 × 0.70 = 3.15 ms    ← dead time посчитана только один раз!
Pulse 1: 4.5 × 0.30 = 1.35 ms
Сумма: 4.5 ms
```

### Стало (ПРАВИЛЬНО):
```
baseFuelMass = 0.4 g

pulse0_mass = 0.4 × 0.70 = 0.28 g
Pulse 0: 0.28 × 10 + 0.5 = 3.3 ms  ← dead time 0.5 ms для этого импульса

pulse1_mass = 0.4 × 0.30 = 0.12 g  
Pulse 1: 0.12 × 10 + 0.5 = 1.7 ms  ← dead time 0.5 ms для этого импульса

Сумма: 3.3 + 1.7 = 5.0 ms ✓
```

## Изменённые файлы

1. `controllers/engine_cycle/multi_injection.cpp`
   - Добавлено `#include "injector_model.h"`
   - Изменена функция `updateMultiInjectionAngles()` чтобы использовать массу вместо длительности
   - Каждый импульс рассчитывается через injectorModel независимо

2. `controllers/settings.cpp`
   - Улучшен вывод `printMultiInjectionAngles()` - теперь показывает массу топлива для отладки

## Как проверить

### Команды консоли:
```
mi_angles   - показывает углы и длительность для каждого цилиндра
mi_tables   - показывает таблицы конфигурации
```

### Осцилоскоп
- Pulse 0 + Pulse 1 должны быть примерно равны одному обычному впрыску
- Или чуть больше на величину одного dead time (что нормально для двух импульсов)

## Результат

После этого исправления:
- ✅ Расчет длительности мультивпрыска стал правильным
- ✅ Dead time применяется корректно к каждому импульсу
- ✅ Сумма двух импульсов примерно равна одному впрыску (±dead time)
- ✅ На осцилоскопе должны увидеть меньшие значения времени
