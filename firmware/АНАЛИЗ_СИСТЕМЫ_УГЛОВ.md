# Анализ системы углов в RUSEFI для Pulse 0 и Pulse 1

## Система координат RUSEFI

RUSEFI использует **абсолютные углы цикла 0-720°**:

```
Цилиндр 1:
0°      90°     180°    270°    360°
|       |       |       |       |
TDC     |      BDC      |      TDC
|       |       |       |       |
Intake  Compress Expan  Exhaust Overlap

Цилиндр 2:
360°    450°    540°    630°    720°
|       |       |       |       |
TDC     |      BDC      |      TDC (next cycle)
```

## BTDC система (Before Top Dead Center)

Таблица `injectionPhase` содержит углы в **BTDC** (опережение впрыска):
- **Положительное значение** = впрыск **ПЕРЕД** TDC (раньше)
- **Отрицательное значение** = впрыск **ПОСЛЕ** TDC (позже)

### Пример:
- BTDC = +534° означает впрыск на 534° перед TDC
- В абсолютной системе это: 720° - 534° = **186° абсолютно**

- BTDC = -100° означает впрыск на 100° после TDC
- В абсолютной системе это: 0° + 100° = **100° абсолютно**

## Как работает computeInjectionAngle() для Pulse 0

```cpp
float injectionOffset = getInjectionOffset(rpm, load);  // Из таблицы injectionPhase
float durationAngle = getInjectionAngleCorrection(fuelMs, oneDegreeUs);  // Длительность
float openingAngle = injectionOffset - durationAngle;  // НАЧАЛО впрыска

return openingAngle;
```

### Вопрос: что такое injectionOffset?

**Гипотеза 1:** injectionOffset это КОНЕЦ впрыска (в BTDC)
- Тогда: `openingAngle = конец - длительность = начало` ✓ Логично!

**Гипотеза 2:** injectionOffset это НАЧАЛО впрыска (в абсолютной системе)
- Тогда: `openingAngle = начало - длительность = ?` ✗ Не имеет смысла!

**Вывод:** injectionOffset это **КОНЕЦ впрыска** (в BTDC)

## Из логов: Pulse 0 = 534°

В логах выводится: `Pulse 0: angle=534.0°`

Это **НАЧАЛО** впрыска (outputChannels->injectionOffset).

Если Pulse 0 начинается в 534°, а длительность 0.8°, то:
- Конец Pulse 0: 534° + 0.8° = 534.8°
- Это соответствует конечному углу из таблицы (~534° BTDC)

**Логика рассчета Pulse 0:**
1. Таблица injectionPhase говорит: "впрыск должен заканчиваться на 534° BTDC"
2. Вычитаем длительность: 534° - 0.8° = 533.2° (начало)
3. **НО** вывод показывает 534°, не 533.2°!

Возможные объяснения:
- Длительность так мала что округляется
- Или вывод использует конец, не начало
- Или в режиме START-injection (начало не сдвигается)

## Система для Pulse 1

Pulse 1 рассчитывается из таблицы `secondInjectionAngleTable`:

```cpp
float baseAngle = interpolate3d(secondInjectionAngleTable, ...);
// secondInjectionAngleTable содержит что?
```

### Вопрос пользователя:

"Нулевая точка у нас в верхней мертвой точке после такта сжатия.
Второй впрыск должен быть рассчитан так: получается что нулевая точка у нас в верхней мертвой точке,
значит второй впыск должен быть ранше нулевой точки на 100 градусов,
значит наше табличное значение показывает опережение.
Получается надо 720 отнять наше табличное значение чтобы получить фазу начала или конца впрыска."

### Перевод на язык кода:

Если таблица `secondInjectionAngleTable` содержит **100°** для Pulse 1:
- Это означает Pulse 1 **на 100° ПЕРЕД TDC**
- В абсолютной системе (0-720°): **720° - 100° = 620°**

**ТЕКУЩИЙ КОД (может быть НЕПРАВИЛЬНО):**
```cpp
float baseAngle = 100;  // Из таблицы
// ... применяется timing mode correction ...
correctedAngle = baseAngle;  // = 100°
return normalizeAngle(correctedAngle);  // = 100°
```

**ПРАВИЛЬНЫЙ КОД (если нужна трансформация):**
```cpp
float baseAngle = 100;  // Из таблицы (BTDC)
// Преобразовать BTDC в абсолютный угол
float absoluteAngle = 720.0f - baseAngle;  // = 620°
// ... применяется timing mode correction ...
return normalizeAngle(absoluteAngle);
```

## Сравнение Pulse 0 vs Pulse 1

### Pulse 0 (обычный впрыск):
- Таблица: `injectionPhase` (BTDC, диапазон -720 до 720)
- Код: `openingAngle = injectionPhase - duration` 
- Результат в логах: 534° (кажется как абсолютный угол или BTDC)

### Pulse 1 (второй впрыск):
- Таблица: `secondInjectionAngleTable` (диапазон 0 до 720)
- Код: `correctedAngle = baseAngle` (+ timing mode correction)
- Результат в логах: 100°

**Вопрос:** Используют ли обе системы одно и то же представление углов?

Если Pulse 0 использует BTDC (534° = 534° BTDC), то 100° в Pulse 1 это тоже BTDC?
Или 100° это абсолютный угол (который в BTDC = 720 - 100 = 620°)?

## Что нужно исправить

1. ✅ Убедиться что Pulse 0 и Pulse 1 используют **одну и ту же систему** представления углов

2. ⏳ Если Pulse 1 таблица содержит BTDC (опережение), но интерпретируется как абсолютный угол:
   - **НУЖНА трансформация: `720 - tableValue`**

3. ⏳ Если обе таблицы содержат разные представления:
   - Нужно документировать это и применять правильные преобразования

## Как проверить

### На консоли:
```
mi_angles
```
Вывести:
- Pulse 0 значение из логов: 534°
- Pulse 1 значение из логов: 100°

Затем проверить в TunerStudio:
- Что написано в таблице injectionPhase для этих условий
- Что написано в таблице secondInjectionAngleTable

И убедиться что преобразования правильны!

## Вывод

**Возможно нужно исправить computeSecondaryInjectionAngle():**

Если таблица содержит BTDC (опережение), но используется как абсолютный угол, то:

```cpp
float baseAngle = interpolate3d(secondInjectionAngleTable, ...);  // BTDC?
if (isTableInBTDC) {
    baseAngle = 720.0f - baseAngle;  // Преобразовать в абсолютный угол
}
// ... дальше применять как обычно ...
```
