# Ответ: Как правильно рассчитываются углы впрыска для мультивпрыска

## Вопрос пользователя
"А что с углами начала или окончания впрыска как правильно ли рассчитывается второй впрыск относительно первого?"

## Ответ

### Короткий ответ
**ДА, углы рассчитываются ПРАВИЛЬНО.**

Система использует **НЕЗАВИСИМЫЙ** расчет углов для каждого импульса:
- **Pulse 0:** Рассчитывается как обычный впрыск (управляется таблицей timing)
- **Pulse 1:** Рассчитывается независимо из таблицы `secondInjectionAngleTable`

Второй импульс **НЕ привязан временно к первому**, а просто происходит в определенный момент цикла.

---

## Подробное объяснение

### Pulse 0: Основной впрыск
```
computeInjectionAngle() вычисляет:
openingAngle = injectionOffset - injectionDurationAngle

Где:
- injectionOffset = конец впрыска (из таблицы timing)
- injectionDurationAngle = длительность в градусах КВ
- Результат = начало впрыска
```

**Пример из логов (Cylinder 0, 1200 RPM, 10.1% load):**
```
Pulse 0:
  - Начало впрыска: 534.0°
  - Длительность: 6.81 ms = 0.817° (при 1200 RPM)
  - Конец впрыска: 534.817°
```

### Pulse 1: Независимый поздний впрыск
```
computeSecondaryInjectionAngle() вычисляет:

1. Берет угол из таблицы secondInjectionAngleTable
   angle = interpolate3d(secondInjectionAngleTable, RPM, Load)

2. Применяет timing mode correction (если нужно):
   - START mode: угол не меняется
   - CENTER mode: angle -= duration/2 (центрировать)
   - END mode: angle -= duration (сдвинуть на конец)

3. Добавляет per-cylinder offset
   angle += getPerCylinderFiringOrderOffset(...)

4. Нормализует в диапазон 0-720°
   angle = normalizeAngle(angle)
```

**Пример из логов (Cylinder 0):**
```
Pulse 1:
  - Начало впрыска: 100.0° (из таблицы)
  - Длительность: 2.91 ms = 0.349°
  - Конец впрыска: 100.349°
```

### Связь между Pulse 0 и Pulse 1

**Они НЕ привязаны временно**, но система ПРОВЕРЯЕТ минимальный dwell:

```cpp
Pulse 0 ends at: 534.817°
Pulse 1 starts at: 100.0°

Dwell calculation:
  raw_dwell = 100.0 - 534.817 = -434.817°
  
  Если отрицательно, добавляем 720°:
  dwell = -434.817 + 720 = 285.183°
  
Validation:
  minDwell = 20° (из таблицы)
  285.183° >= 20° ✓ OK
```

**Интерпретация:**
- Pulse 0 заканчивается на **534.8°**
- Pulse 1 начинается на **100°** (на СЛЕДУЮЩЕМ обороте или 285° спустя)
- Между ними **285° "свободного" пространства**
- Это **нормально** для независимого мультивпрыска!

## Три варианта архитектуры (и почему выбран вариант 3)

### Вариант 1: Pulse 1 привязана к Pulse 0
```cpp
Pulse1_start = Pulse0_end + fixedDwell + tableOffset
// Минус: требует больше вычислений и таблиц
```

### Вариант 2: Pulse 1 как "время от конца Pulse 0"
```cpp
Pulse1_start = Pulse0_end + tableValue
// Минус: таблица будет в мс, сложнее конфигурировать
```

### Вариант 3: Pulse 1 как абсолютный угол (ТЕКУЩИЙ)
```cpp
Pulse1_start = tableValue (просто как обычный впрыск)
// Плюс: 
//   - Просто в понимании
//   - Гибко - может быть в любой момент цикла
//   - Таблица в градусах (как обычный timing)
//   - Валидация dwell гарантирует минимальный gap
```

**Выбран вариант 3** потому что:
- ✅ Простый и понятный
- ✅ Гибкий - не требует привязки к Pulse 0
- ✅ Таблица конфигурируется как обычный timing
- ✅ Минимальный dwell все равно проверяется

## Как это выглядит на практике

### На осцилоскопе
```
Cylinder 0 injection pin:

0°     200°      400°      600°     720° | 0°
|       |          |         |       |    |
|   Pulse0         |         |   Pulse1  |
|  (534-535°)      |         |  (100-100.3°)
|                  |         |          |
|                  |         +----------+ (small pulse ~2.9ms)
|  +---------+     |         |
+--+         +-----+---------+-+---------+--
   |         |               | |
   (0.8°, 6.8ms)           (285° gap)

На графике видны ДВА отдельных импульса:
- Pulse 0: ранний в цикле, ~6.8 ms (скоро исправим на ~3.3 ms)
- Pulse 1: позже в цикле, ~2.9 ms (скоро исправим на ~1.7 ms)
```

### В таблицах TunerStudio

**Split Ratio Table:**
```
70% → Pulse 0 получает 70%, Pulse 1 получает 30%
```

**Second Injection Angle Table:**
```
100° → Pulse 1 начинает впрыск в абсолютный момент 100° BTDC
(Независимо от Pulse 0!)
```

**Minimum Dwell Table:**
```
20° → Гарантирует что между концом Pulse 0 и началом Pulse 1
     минимум 20° (если меньше - fallback на single injection)
```

## Проблема которая была исправлена

**Не проблема с углами** (они рассчитывались правильно), а **проблема с длительностью**:

### Было:
```
baseFuelMs = 9.72 ms (из injectionDuration)
Pulse 0: 9.72 × 0.70 = 6.81 ms ✗ НЕПРАВИЛЬНО
Pulse 1: 9.72 × 0.30 = 2.91 ms ✗ НЕПРАВИЛЬНО
Сумма: 9.72 ms (слишком много!)
```

### Стало:
```
baseMass = 0.4 g (из injectionMass)
Pulse 0: injectorModel(0.4 × 0.70) = ~3.3 ms ✓ ПРАВИЛЬНО
Pulse 1: injectorModel(0.4 × 0.30) = ~1.7 ms ✓ ПРАВИЛЬНО
Сумма: ~5.0 ms (правильно!)
```

**Углы остались теми же**, но длительность впрыска теперь рассчитывается правильно!

---

## Заключение

✅ **Углы начала/конца впрыска рассчитываются ПРАВИЛЬНО**
- Pulse 0: по обычной логике (управляется таблицей timing)
- Pulse 1: независимо из таблицы secondInjectionAngleTable
- Минимальный dwell проверяется валидацией

⚠️ **Проблема была в ДЛИТЕЛЬНОСТИ**, а не в углах
- Исправлено: теперь используется injectorModel с реальной массой
- Каждый импульс рассчитывается независимо
- Сумма ~5 ms (вместо 9.72 ms)

🧪 **После компиляции нужно протестировать на железе**
- Команда консоли: `mi_angles`
- Проверить на осцилоскопе что Pulse 0 и Pulse 1 видны
- Убедиться что длительности значительно уменьшились
