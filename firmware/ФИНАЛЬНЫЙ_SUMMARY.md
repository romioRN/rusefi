## Исправление #1: Расчет ДЛИТЕЛЬНОСТИ впрыска ✅

### Проблема
Pulse 0: 6.81 ms, Pulse 1: 2.91 ms → **Сумма: 9.72 ms** (НЕПРАВИЛЬНО, должна ~5 ms)

### Причина
Линейное масштабирование уже рассчитанной длительности вместо использования реальной массы.

### Решение
```cpp
// Было:
float baseFuelMs = getEngineState()->injectionDuration;  // 9.72 ms
float pulse0Ms = baseFuelMs * 0.70;  // 6.81 ms ✗

// Стало:
float baseMass = getEngineState()->injectionMass[cyl];
float pulse0Mass = baseMass * 0.70;
float pulse0Ms = injectorModel->getInjectionDuration(pulse0Mass);  // 3.3 ms ✓
```

### Файлы
- `controllers/engine_cycle/multi_injection.cpp` (функция `updateMultiInjectionAngles()`)

### Результат
- Pulse 0: 6.81 ms → ~3.3 ms (↓ 52%)
- Pulse 1: 2.91 ms → ~1.7 ms (↓ 42%)
- **Сумма: 9.72 ms → ~5.0 ms (↓ 49%)** ✓

## Исправление #2: Преобразование BTDC для Pulse 1 ✅

### Проблема
Таблица `secondInjectionAngleTable` содержит углы в **BTDC** (опережение впрыска), но интерпретировались как абсолютные углы цикла.

Пример:
- Таблица содержит: 100°
- Интерпретировалось как: 100° абсолютно ✗
- Должно быть: 100° BTDC → 720° - 100° = **620° абсолютно** ✓

### Причина
Отсутствовала трансформация BTDC → абсолютный угол для Pulse 1.

Для сравнения, Pulse 0 это правильно обрабатывает в `computeInjectionAngle()`:
```cpp
angle_t openingAngle = injectionOffset - injectionDurationAngle;
// Автоматически преобразуется в правильную систему координат
```

### Решение
```cpp
// Было:
float baseAngle = 100;  // Интерпретировано неправильно ✗

// Стало:
float btdcAngle = 100;  // Из таблицы (BTDC)
float baseAngle = 720.0f - btdcAngle;  // = 620° абсолютно ✓
```

### Подтверждение из конфига
```ini
upDownLabel = "(later) Angle BTDC (sooner)"
```

Этот лейбл ясно указывает что таблица содержит **BTDC углы**.

### Файлы
- `controllers/engine_cycle/multi_injection.cpp` (функция `computeSecondaryInjectionAngle()`)

### Результат логов
```
Было:
Pulse 0: angle=534.0°, Pulse 1: angle=100.0°

Стало:
Pulse 0: angle=534.0°, Pulse 1: angle=620.0°  ✓

Dwell между ними: 620 - 534.8 = 85.2° (вместо неправильных 285.2°)
```

## 🎓 Важные выводы

### Система углов в RUSEFI

**RUSEFI использует две связанные системы:**

1. **Таблица `injectionPhase`**: Содержит **BTDC** углы
   - Положительные значения = впрыск ДО TDC (раньше)
   - Отрицательные значения = впрыск ПОСЛЕ TDC (позже)
   - Преобразуется в абсолютный угол автоматически

2. **Таблица `secondInjectionAngleTable`**: Также содержит **BTDC** углы
   - ~~Раньше НЕ преобразовывались~~ (БАГ)
   - Теперь преобразуются через: `baseAngle = 720 - btdcAngle`

### Длительность впрыска

**Правильная логика:**
1. Получить реальную **массу** топлива для цилиндра
2. Разделить массу по **ratio** между импульсами
3. Рассчитать **длительность** каждого импульса через injectorModel

**Неправильная логика:**
1. ~~Получить базовую длительность впрыска~~
2. ~~Разделить длительность по ratio~~

Первый способ правильный потому что учитывает **dead time** каждого импульса!
