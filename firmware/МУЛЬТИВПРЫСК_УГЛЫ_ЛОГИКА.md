# Мультивпрыск: Как работает расчет углов

## Архитектура мультивпрыска

Система поддерживает **два независимых впрыска** (pulse) в один цикл двигателя:

### Pulse 0: Основной впрыск
- **Угол:** Рассчитывается как обычный впрыск через `computeInjectionAngle()`
- **Логика:** `openingAngle = injectionOffset - injectionDurationAngle`
  - `injectionOffset` - конец впрыска (BTDC, управляется обычной таблицей)
  - `injectionDurationAngle` - длительность впрыска в градусах
  - Результат - начало впрыска

### Pulse 1: Поздний впрыск
- **Угол:** Читается из таблицы `secondInjectionAngleTable`
- **Логика:** Независимый впрыск, может быть в любой момент цикла
- **Таблица:** 3D таблица по RPM × Load, содержит абсолютный угол BTDC

## Пример из логов (1200 RPM, 10.1% load)

```
Onze Degree: 8.33 ms/deg (60,000,000 / (1200 * 6))

Pulse 0:
  - Angle: 534° (начало впрыска из computeInjectionAngle)
  - Duration: 6.81 ms = 0.817° (очень мало, потому что 1200 RPM медленно)
  - End angle: 534.817°

Pulse 1:
  - Angle: 100° (из таблицы secondInjectionAngleTable)
  - Duration: 2.91 ms = 0.349°
  - End angle: 100.349°

Dwell между ними:
  - 100° - 534.817° = -434.817°
  - Нормализуем: -434.817° + 720° = 285.183°
  - Минимум требуемый: 20° (из таблицы minDwellAngleTable)
  - Проверка: 285.183° >= 20° ✓ OK
```

**Интерпретация:**
- Pulse 0 заканчивается на 534.8°
- Pulse 1 начинается на 100° (на следующем обороте или на "100 градусов вперед")
- Between them: 285° свободного пространства
- Это **нормально** для мультивпрыска с ранним Pulse 0 и поздним Pulse 1

## Расчет length (после исправления)

### Было (НЕПРАВИЛЬНО)
```cpp
float baseFuelMs = getEngineState()->injectionDuration;  // 9.72 ms (рассчитано один раз)
float pulse0Ms = baseFuelMs * 0.70;  // 6.81 ms
float pulse1Ms = baseFuelMs * 0.30;  // 2.91 ms
// Проблема: каждый импульс имеет НЕПРАВИЛЬНЫЙ dead time!
```

### Стало (ПРАВИЛЬНО)
```cpp
float baseMass = getEngineState()->injectionMass[cyl];  // 0.4 g (реальная масса)

float pulse0Mass = baseMass * 0.70;  // 0.28 g
float pulse0Ms = injectorModel->getInjectionDuration(pulse0Mass);  // 3.3 ms

float pulse1Mass = baseMass * 0.30;  // 0.12 g
float pulse1Ms = injectorModel->getInjectionDuration(pulse1Mass);  // 1.7 ms

// Total: 3.3 + 1.7 = 5.0 ms ✓ (примерно как обычный впрыск)
```

## Режимы впрыска (Timing Modes)

Система поддерживает три режима для Pulse 0:
- **START:** Впрыск начинается в указанный момент
- **CENTER:** Впрыск центрирован в указанный момент
- **END:** Впрыск заканчивается в указанный момент

Для Pulse 1 применяется аналогичная коррекция на основе таблицы!

## Валидация перекрытия

Функция `validateInjectionWindows()` проверяет что:
```cpp
dwell = Pulse1_start - Pulse0_end
if (dwell < 0) dwell += 720;
return (dwell >= minDwellAngleTable);
```

Если dwell меньше минимума - система fallback на single injection.

## Осцилоскоп проверка

На осцилоскопе при мультивпрыске должны видеть:

1. **Pulse 0:** На графике Pulse 0 (Injection 1)
   - Должен быть в районе 534° при 1200 RPM
   - Длительность должна быть ~3-4 ms (после исправления с массой)

2. **Pulse 1:** На графике Pulse 1 (Injection 2)
   - Должен быть в другой части цикла (около 100°)
   - Длительность должна быть ~1-2 ms (после исправления с массой)

3. **Полная сумма:** Оба импульса вместе должны быть ~5 ms
   - Примерно равны одному обычному впрыску

## Конфигурация таблиц

### multiInjectionSplitRatioTable
- Процент топлива для Pulse 0 (Pulse 1 получает 100 - этот %)
- Например: 70 означает 70% на Pulse 0, 30% на Pulse 1

### secondInjectionAngleTable
- Абсолютный угол BTDC для Pulse 1
- **НЕЗАВИСИМО** от Pulse 0
- Может быть в любой точке 0-720°

### minDwellAngleTable
- Минимальный dwell (зазор) между концом Pulse 0 и началом Pulse 1
- По умолчанию 20°
- Если нарушается - система переходит на single injection
