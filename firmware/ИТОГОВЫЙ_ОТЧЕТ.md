# ИТОГОВЫЙ ОТЧЕТ: Исправление расчета мультивпрыска

## 📋 Краткое резюме

**Найдена и исправлена критическая проблема в расчете ДЛИТЕЛЬНОСТИ мультивпрыска.**

### Симптомы
- Pulse 0: 6.81 ms (70%)
- Pulse 1: 2.91 ms (30%)
- **Сумма: 9.72 ms** ← **БЫЛО НЕПРАВИЛЬНО, ИСПРАВЛЕНО**

### Корневая причина
Система использовала **линейное масштабирование** уже рассчитанной длительности впрыска, вместо использования реальной **массы топлива** и расчета каждого импульса через injectorModel.

### Решение
Переход с масштабирования длительности на масштабирование МАССЫ с независимым расчетом каждого импульса через injectorModel.

## 🔧 Технические изменения

### Файл: `controllers/engine_cycle/multi_injection.cpp`

**Добавлено:**
```cpp
#include "injector_model.h"
```

**Изменено в `updateMultiInjectionAngles()`:**

#### Было (НЕПРАВИЛЬНО):
```cpp
floatms_t baseFuelMs = getEngineState()->injectionDuration;
for (uint8_t i = 0; i < numberOfPulses; i++) {
    float ratio = computeSplitRatio(i);
    floatms_t pulseFuelMs = baseFuelMs * (ratio / 100.0f);  // ✗ Линейное масштабирование
    pulses[i].fuelMs = pulseFuelMs;
}
```

#### Стало (ПРАВИЛЬНО):
```cpp
float baseFuelMass = getEngineState()->injectionMass[cylinderNumber];
for (uint8_t i = 0; i < numberOfPulses; i++) {
    float ratio = computeSplitRatio(i);
    float pulseMass = baseFuelMass * (ratio / 100.0f);  // ✓ Масштабируем МАССУ
    
    // ✓ Рассчитываем длительность КАЖДОГО импульса через injectorModel
    floatms_t pulseFuelMs = engine->module<InjectorModelPrimary>()->getInjectionDuration(pulseMass);
    
    pulses[i].fuelMs = pulseFuelMs;
    pulses[i].splitRatio = ratio;
}
```

### Файл: `controllers/settings.cpp`

**Улучшено в `printMultiInjectionAngles()`:**

Добавлен вывод массы топлива для отладки:
```cpp
float fuelMass = getEngineState()->injectionMass[i];
float pulseMass = fuelMass * (pulse.splitRatio / 100.0f);
efiPrintf("  Pulse %d: ... mass=%.3f g", pulseIdx, pulseMass);
```

## 📊 Ожидаемые результаты

### До исправления:
```
injectionMass = 0.4 g

Pulse 0: 4.5 × 0.70 = 3.15 ms
Pulse 1: 4.5 × 0.30 = 1.35 ms
Сумма: 4.5 ms ← НЕПРАВИЛЬНО (dead time потерян)
```

### После исправления:
```
baseFuelMass = 0.4 g

pulseMass0 = 0.4 × 0.70 = 0.28 g
Pulse 0: injectorModel(0.28) = 3.3 ms  ← Правильный dead time
        (если модель: dur = mass × 10 + 0.5)

pulseMass1 = 0.4 × 0.30 = 0.12 g
Pulse 1: injectorModel(0.12) = 1.7 ms  ← Правильный dead time

Сумма: 3.3 + 1.7 = 5.0 ms ✓ ПРАВИЛЬНО
```

## 🎯 Что было ПРАВИЛЬНО (не требовало исправлений)

✅ **Углы впрыска (начало/конец)**
- Pulse 0 рассчитывается по обычной логике (computeInjectionAngle)
- Pulse 1 рассчитывается независимо из таблицы secondInjectionAngleTable
- Минимальный dwell проверяется валидацией

✅ **Распределение топлива (split ratio)**
- Pulse 0: получает значение из таблицы (например, 70%)
- Pulse 1: получает 100 - значение (например, 30%)
- Логика правильная, просто было применено к неправильной базе

✅ **Валидация перекрытия (dwell)**
- Проверяется минимальный зазор между концом Pulse 0 и началом Pulse 1
- Если нарушается - fallback на single injection
